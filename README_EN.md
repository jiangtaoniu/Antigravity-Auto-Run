# Antigravity Auto Run

**Smart IDE Permission Auto-Approval Agent** — for Antigravity / VS Code / Cursor

**[🌐 中文文档](./README.md)**

---

## Overview

Auto Approve Agent is a VS Code-compatible extension that automatically handles all permission confirmation dialogs generated by AI Agents during runtime. It implements a **5-layer automation engine + dual safety filter** architecture to achieve millisecond-level auto-response for 50+ button variants (`Allow`, `Run`, `Retry`, etc.) without disturbing the user's foreground workflow.

> **Use case**: When an AI Agent (e.g., Gemini) continuously performs file operations, command execution, and network requests in the background, the IDE generates frequent security confirmation dialogs. This extension eliminates all manual intervention, enabling fully autonomous Agent operation.

## Architecture

### 5-Layer Automation Engine

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: Toast Notification Poller (Extension Host)    │
│  Calls notifications.acceptPrimaryAction every 500ms    │
├─────────────────────────────────────────────────────────┤
│  Layer 2: UIAutomation Silent Invoke                    │
│  InvokePattern / LegacyIAccessiblePattern — zero focus  │
├─────────────────────────────────────────────────────────┤
│  Layer 3: user32.dll Keyboard Injection + Idle Guard    │
│  Physical Alt+Enter injection, guarded by 3-tier logic  │
├─────────────────────────────────────────────────────────┤
│  Layer 4: Warmup Cold-Start Protection                  │
│  First scan caches existing buttons, prevents jitter    │
├─────────────────────────────────────────────────────────┤
│  Layer 5: Base64 UTF-16LE Encoding                      │
│  Cross-locale encoding, prevents CJK path corruption    │
└─────────────────────────────────────────────────────────┘
```

**Layer 1 — Toast Poller**: Accepts notification toast primary actions via the VS Code Extension Host API. Electron toasts are invisible to Windows UIAutomation — this layer bypasses from within.

**Layer 2 — Silent UI Automation**: A PowerShell background process scans all button controls inside IDE windows, attempting `InvokePattern` and `LegacyIAccessiblePattern` first — both are completely silent API triggers. **Zero focus stealing.**

**Layer 3 — Keyboard Injection + Smart Idle Guard**: When Layer 2 is blocked by Shadow DOM, falls back to `user32.dll` physical `Alt+Enter` injection. Before injecting, it checks real user activity via `GetLastInputInfo` API:

| Condition | Action |
|---|---|
| User is in the IDE | ✅ Inject immediately |
| User elsewhere + idle > 10s | ✅ AFK detected, safe to inject |
| User elsewhere + button pending > 100s | ✅ Timeout, force inject |
| User actively typing elsewhere | ⏳ Skip, retry next cycle |

After injection, `StealthAltEnter` automatically restores the user's previous foreground window (~250ms).

**Layer 4 — Warmup Cold-Start Protection**: On IDE restart, the first scan silently caches all existing buttons' `RuntimeId` without clicking or scrolling. New buttons are only processed from the second cycle onward.

**Layer 5 — Encoding Shield**: Compiles PowerShell commands into Base64 UTF-16LE payloads, executed via `-EncodedCommand`. Mathematically eliminates CJK path corruption in Node.js `CreateProcess` calls.

### Dual Safety Filter System

Replaces traditional single-regex matching with a **blacklist-first + whitelist fuzzy-match** architecture:

#### 🚫 Blacklist (Absolute Priority)

Buttons containing ANY of these keywords are **never clicked**:

```
deny, reject, cancel, decline, delete, remove, disable, block, stop, abort,
don't, do not, never, revoke, sign out, log out, uninstall, close, exit, quit,
dismiss, discard, revert, rollback, undo, skip, ignore
(+ Chinese equivalents)
```

#### ✅ Whitelist (Tiered Fuzzy Matching)

After passing the blacklist, buttons matching any of these patterns are auto-clicked:

| Priority | Keywords | Example Matches |
|---|---|---|
| Tier 1 | `allow`, `accept`, `approve`, `authorize`, `permit`, `grant` | Allow This Conversation, Accept All, Grant Access |
| Tier 2 | `run`, `continue`, `proceed`, `confirm`, `yes`, `ok`, `trust`, `retry`, `always` | Run Command, Continue Anyway, Yes to All |
| Tier 3 | `enable`, `save`, `submit`, `execute`, `apply`, `install`, `update`, `reload`, `restart` | Reload Window, Apply Changes |
| Chinese | `允许`, `确认`, `确定`, `运行`, `继续`, `重试`, `信任`, `接受`, `授权`... | 允许本次会话, 总是运行 |

## Installation

### Requirements
- **Windows 10 / 11** (requires native `powershell.exe`)
- VS Code / Antigravity / Cursor or any Electron-based IDE

### Install

**Option 1: From Marketplace** (Recommended)

Search `Antigravity Auto Run` in the IDE Extensions panel and install directly.

**Option 2: From VSIX**

1. Download the latest `.vsix` from [Releases](../../releases)
2. IDE → `Ctrl+Shift+X` → `...` menu → "Install from VSIX..."
3. **Restart IDE**

### Usage

The extension runs **automatically with zero configuration**. For manual control:

- `Ctrl+Shift+P` → `Antigravity Auto Run: Start`
- `Ctrl+Shift+P` → `Antigravity Auto Run: Stop`

> ⚠️ Do not minimize the IDE to the taskbar (Chromium suspends the accessibility tree). Instead, leave the IDE window open on desktop and cover it with your browser/game in full screen.
